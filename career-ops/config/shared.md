# Career-Ops Shared Context

Read this file before any evaluation. It defines how jobs are scored, what archetypes exist, and the ethical rules that apply to all career-ops skills.

---

## Scoring Rubric

Every job is scored 1.0-5.0 across 6 dimensions. The weighted average determines the overall score.

| Dimension | Weight | What It Measures |
|-----------|--------|-----------------|
| CV Match | 25% | How well Erik's skills and experience align with the JD requirements |
| North Star | 20% | How well the role fits Erik's target archetype and career direction |
| Compensation | 20% | Whether the salary range matches Erik's target (see profile.yml) |
| Culture Signals | 15% | Remote policy, team size, growth stage, values alignment |
| Red Flags | 10% | Blockers: unrealistic requirements, bad Glassdoor, vague JD, churn signals |
| Growth Potential | 10% | Career trajectory fit, learning opportunity, title progression |

### Scoring Scale

- **5.0**: Perfect fit. Every box checked.
- **4.0-4.9**: Strong fit. Minor gaps only.
- **3.0-3.9**: Decent fit. Notable gaps but worth considering.
- **2.0-2.9**: Weak fit. Major misalignment.
- **1.0-1.9**: Poor fit. Don't waste time.

### Recommendation Thresholds

| Score | Action |
|-------|--------|
| 4.5+ | Apply now. Prioritize this one. |
| 4.0-4.4 | Strong candidate. Worth applying. |
| 3.5-3.9 | Only if there's a specific reason (network connection, dream company, unique role). |
| Below 3.5 | Skip. Don't apply. |

## Evaluation Report Format (Blocks A-F)

Every evaluation produces 6 blocks saved as a markdown report.

### Block A: Role Summary
Table with: Company, Role Title, Location, Remote Policy, Detected Archetype, Overall Score, Recommendation

### Block B: CV Match Analysis
Table comparing each JD requirement against Erik's experience. Columns: Requirement, Erik's Match, Gap Level (None/Minor/Major), Notes

### Block C: Level & Strategy
Seniority assessment. Is this role a step up, lateral, or step down? How should Erik position himself? What's the framing strategy for this archetype?

### Block D: Compensation Research
Use web search to find salary data for this role + company + location. Compare against Erik's target range. Flag if below minimum.

### Block E: CV Customization Plan
Which sections of cv.md to emphasize, which keywords from the JD to weave in, what to de-emphasize. Specific, actionable.

### Block F: Interview Strategy
2-4 STAR+R stories from story-bank.md (or new ones to add) mapped to likely interview questions. Red flag questions to prepare for. Case study suggestions.

## Ethical Rules

These are non-negotiable. Every skill must follow them.

1. **Never invent metrics or claims.** Everything in a CV, cover letter, or application must come from cv.md or profile.yml. If it's not there, don't write it.
2. **Never auto-submit applications.** Always present to Erik for review first. The apply skill generates answers but never clicks submit.
3. **Never apply below threshold.** If the score is below 3.5, don't generate a CV or application. Flag it as a skip.
4. **Never spam.** Scanning is for finding good fits, not mass-applying. Quality over quantity.
5. **Anti-AI writing rules apply.** All generated text (cover letters, form answers, LinkedIn messages) must follow Erik's global anti-AI writing rules. No banned words. No banned patterns. Contractions always.
6. **Cite your sources.** When referencing Erik's experience in generated text, note which cv.md section or proof point it came from.
